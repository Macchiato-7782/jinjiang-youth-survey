## 二、遇到的问题与教训

### 2.1 问题清单

| 序号 | 问题描述 | 原因 | 解决方案 |
|------|---------|------|---------|
| 1 | npm install 报错 "Invalid Version" | package.json 缺少 engines 字段，Node 版本不兼容 | 手动安装 Node.js 18 |
| 2 | 服务器启动后立即退出 | server.js 缺少 `app.listen()` 启动代码 | 添加服务器监听代码 |
| 3 | Node.js 版本太旧 (v12) | 系统默认 apt 源版本过旧 | 手动下载 Node.js 18 二进制包 |
| 4 | sqlite3 编译卡住 | 需要编译原生模块，耗时很长 | 使用 yarn 替代 npm 安装 |
| 5 | dpkg 损坏报错 | 之前安装过程中断导致 | 执行 `dpkg --configure -a` 修复 |
| 6 | npm 缓存损坏 | 多次安装失败导致 | 清除 npm 缓存 |
| 7 | GitHub 推送冲突 | 本地和远程版本不一致 | 强制推送或合并分支 |

### 2.2 关键教训

1. **Node.js 版本必须 16+**
   - 项目依赖 sqlite3 需要较新的 Node.js
   - Ubuntu 默认 apt 源的 Node.js 12 太旧，必须手动升级

2. **package.json 必须完整**
   - 所有依赖（express, sqlite3, cors, body-parser, uuid）必须正确声明
   - 缺少任何一个都会导致启动失败

3. **server.js 必须包含启动代码**
   - `app.listen(PORT, ...)` 是必需的
   - 缺少这个服务器不会真正启动

4. **服务器环境优先用 yarn**
   - yarn 比 npm 更稳定，尤其在安装 sqlite3 时
   - npm 经常卡在编译步骤

5. **不要用 Vercel 部署 Node.js + SQLite**
   - Vercel 是无状态架构，不支持 SQLite 写入
   - 必须用云服务器（腾讯云/阿里云等）

#### 步骤3：一键部署脚本（完整版）

```bash
#!/bin/bash
set -e

echo "========== 开始部署晋江青年婚育观念调查系统 =========="

# 1. 更新系统
echo "[1/7] 更新系统..."
sudo apt update

# 2. 安装Node.js 18（手动安装，避免apt源版本过旧）
echo "[2/7] 安装Node.js 18..."
cd ~
wget https://nodejs.org/dist/v18.20.5/node-v18.20.5-linux-x64.tar.xz
tar -xf node-v18.20.5-linux-x64.tar.xz
sudo mv node-v18.20.5-linux-x64 /usr/local/node18
sudo ln -sf /usr/local/node18/bin/node /usr/bin/node
sudo ln -sf /usr/local/node18/bin/npm /usr/bin/npm
sudo ln -sf /usr/local/node18/bin/npx /usr/bin/npx
node -v  # 验证

# 3. 安装Git和Yarn
echo "[3/7] 安装Git和Yarn..."
sudo apt install -y git
curl -o- -L https://yarnpkg.com/install.sh | bash
export PATH="$HOME/.yarn/bin:$HOME/.config/yarn/global/node_modules/.bin:$PATH"

# 4. 克隆项目
echo "[4/7] 下载项目代码..."
cd /opt
sudo rm -rf jinjiang-youth-survey 2>/dev/null || true
sudo git clone https://github.com/Macchiato-7782/jinjiang-youth-survey.git
cd jinjiang-youth-survey
sudo chmod 777 .  # 确保有写入权限

# 5. 安装依赖（使用yarn，更稳定）
echo "[5/7] 安装依赖..."
sudo /usr/local/node18/bin/yarn install

# 6. 创建数据库目录
echo "[6/7] 配置数据库..."
sudo mkdir -p database
sudo chmod 777 database

# 7. 启动服务
echo "[7/7] 启动服务..."
sudo /usr/local/node18/bin/node server.js &

# 获取IP
IP=$(curl -s ip.sb)

echo ""
echo "=========================================="
echo "✅ 部署完成！"
echo "=========================================="
echo "📝 问卷地址：http://$IP:3000"
echo "📊 后台地址：http://$IP:3000/admin"
echo "=========================================="
```

#### 步骤4：配置防火墙

#### 步骤5：使用PM2持久运行（推荐）
```bash
# 安装PM2
sudo /usr/local/node18/bin/npm install -g pm2

# 停止当前运行
sudo pkill -f "node server.js"

# 用PM2启动
cd /opt/jinjiang-youth-survey
sudo /usr/local/node18/bin/pm2 start server.js --name "survey"
sudo /usr/local/node18/bin/pm2 save
sudo /usr/local/node18/bin/pm2 startup

# 查看状态
sudo /usr/local/node18/bin/pm2 status
sudo /usr/local/node18/bin/pm2 logs survey
```

#### 步骤6：配置Nginx反向代理（可选）
```bash
# 安装Nginx
sudo apt install -y nginx

# 配置
sudo tee /etc/nginx/sites-available/survey << 'EOF'
server {
    listen 80;
    server_name _;
    
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_cache_bypass $http_upgrade;
    }
}
EOF

sudo rm -f /etc/nginx/sites-enabled/default
sudo ln -sf /etc/nginx/sites-available/survey /etc/nginx/sites-enabled/survey
sudo nginx -t && sudo systemctl restart nginx
```

#### 步骤7：绑定域名和HTTPS（可选）
```bash
# 安装Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书（替换为你的域名）
sudo certbot --nginx -d yourdomain.com

# 自动续期已配置
```
